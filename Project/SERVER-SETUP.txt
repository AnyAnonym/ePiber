2. VERSIONIERUNG
================================================================================

Die Versionsnummer wird an EINER einzigen Stelle gepflegt:

  Backend/package.json → "version": "3.0.0"

Das Backend stellt sie über den HTTP-Endpoint /version bereit:

  GET /version → {"version":"3.0.0"}

Das Frontend (global.js) holt die Version per fetch("/version") und zeigt
sie im Footer an. Kein Hardcoding im Frontend nötig.

Versionsänderung: Nur Backend/package.json editieren, committen, deployen.

5. ERSTEINRICHTUNG AUF DEM SERVER
================================================================================

5.1 Pakete installieren
-----------------------
  pacman -S nodejs npm caddy git

5.2 Benutzer anlegen
--------------------
  useradd -r -s /bin/nologin piber

5.3 Verzeichnisse anlegen
--------------------------
  mkdir -p /srv/http/ePiber/{piber,paj,pk}
  chown -R piber:piber /srv/http/ePiber

5.4 Git-Repository klonen (3x)
-------------------------------
  cd /srv/http/ePiber/piber
  git clone <REPO_URL> .

  cd /srv/http/ePiber/paj
  git clone <REPO_URL> .

  cd /srv/http/ePiber/pk
  git clone <REPO_URL> .

5.5 Backend Dependencies installieren (3x)
-------------------------------------------
  cd /srv/http/ePiber/piber/Backend && npm install --production
  cd /srv/http/ePiber/paj/Backend && npm install --production
  cd /srv/http/ePiber/pk/Backend && npm install --production

5.6 Konfigurationsdateien erstellen (pro System!)
--------------------------------------------------

  # --- LIVE (piber) ---

  SHEET_ID=1E1CYezDcScIBvH9ebjN0hOkvttTdA6PFIgYKDMaeE04

  COURT_URL=https://scorer-tennis.b-cdn.net/json/24.voll.json
  GOOGLE_APPLICATION_CREDENTIALS=/srv/http/ePiber/piber/Backend/service-account.json

  # Frontend/JS/SDK.js:
  export const SCORER_WS_URL = "wss://epiber.at/ws";

5.7 Google Service-Account-Key
------------------------------
  Den JSON-Key vom Google Cloud Projekt "e-piber" herunterladen:
    Google Cloud Console → IAM → Service Accounts → Keys → Add Key → JSON

  Die Datei als "service-account.json" in jedes Backend/ legen:
    /srv/http/ePiber/piber/Backend/service-account.json
    /srv/http/ePiber/paj/Backend/service-account.json
    /srv/http/ePiber/pk/Backend/service-account.json

  WICHTIG: Der Service Account braucht Bearbeiter-Zugriff auf alle
  3 Google Spreadsheets!

5.8 Caddyfile installieren
---------------------------
  cp /srv/http/ePiber/piber/Project/Caddyfile /etc/caddy/Caddyfile
  systemctl enable --now caddy

5.9 Systemd-Services installieren
----------------------------------
  cp /srv/http/ePiber/piber/Project/systemd/*.service /etc/systemd/system/
  systemctl daemon-reload
  systemctl enable --now epiber-piber
  systemctl enable --now epiber-paj
  systemctl enable --now epiber-pk

5.10 Firewall öffnen
---------------------
  # Ports 443 (HTTPS), 80 (HTTP Redirect), 8081, 8082 müssen offen sein
  # Bei iptables:
  iptables -A INPUT -p tcp --dport 80 -j ACCEPT
  iptables -A INPUT -p tcp --dport 443 -j ACCEPT
  iptables -A INPUT -p tcp --dport 8081 -j ACCEPT
  iptables -A INPUT -p tcp --dport 8082 -j ACCEPT

  # Bei firewalld:
  firewall-cmd --permanent --add-port=80/tcp
  firewall-cmd --permanent --add-port=443/tcp
  firewall-cmd --permanent --add-port=8081/tcp
  firewall-cmd --permanent --add-port=8082/tcp
  firewall-cmd --reload

5.11 DNS
--------
  A-Record: epiber.at → <Server-IP>
  (Caddy holt sich automatisch ein Let's Encrypt Zertifikat)


================================================================================
6. DEPLOYMENT-WORKFLOW (nach Ersteinrichtung)
================================================================================

Entwicklung (lokal auf Windows, R:\):
  1. Code ändern
  2. git add + git commit + git push

Server-Update:
  # Live-System
  cd /srv/http/ePiber/piber && git pull
  cd Backend && npm install --production    # nur wenn package.json geändert
  systemctl restart epiber-piber            # nur wenn Backend-Code geändert

  # PAJ-Testsystem
  cd /srv/http/ePiber/paj && git pull
  cd Backend && npm install --production
  systemctl restart epiber-paj

  # PK-Testsystem
  cd /srv/http/ePiber/pk && git pull
  cd Backend && npm install --production
  systemctl restart epiber-pk

  Hinweis: Frontend-Änderungen brauchen keinen Restart —
  Caddy liefert die Dateien direkt aus dem Dateisystem.
  Backend-Änderungen erfordern einen Restart des Services.
